# REVIEW.md

## สิ่งสำคัญสำหรับ Repository นี้ (What matters in this repository)
- **ความปลอดภัยสูงสุด (Zero unsafe code):** ห้ามมิให้มีการใช้คีย์เวิร์ด `unsafe` หรือเขียนบล็อกคำสั่งที่ไม่ปลอดภัยในโครงการโดยเด็ดขาด
- **การจัดการข้อผิดพลาดที่เป็นภาษาไทย:** ข้อความแสดงข้อผิดพลาดทั้งหมดต้องเขียนเป็นภาษาไทย และใช้รหัสข้อผิดพลาดกลุ่ม `E1xx` ถึง `E6xx` ตามข้อกำหนดใน [AGENTS.md](file:///D:/dev/10/poe-sdk-rs-fix-pr10/AGENTS.md)
- **ความเข้ากันได้ของรุ่น Rust (MSRV):** โค้ดทั้งหมดต้องรองรับและคอมไพล์ผ่านบน Rust 1.90.0 ซึ่งมีกระบวนการตรวจสอบในระบบ CI
- **เอกสารและตัวอย่างการใช้งาน:** ฟังก์ชันระดับสาธารณะ (Public APIs) ทั้งหมดต้องมีเอกสารอธิบายพร้อมตัวอย่างโค้ด (Doc Comments) ที่สามารถรัน doc-test ผ่าน

## การประเมินระดับความรุนแรง (Severity calibration)
- **Critical (วิกฤต):** โค้ดที่ก่อให้เกิด Stack Overflow, การวนลูปไม่สิ้นสุด (Infinite Loop), ปัญหาหน่วยความจำรั่วไหล (Memory Leak), หรือการปิดแจ้งเตือนคำเตือน Clippy (`#[allow(...)]`) โดยไม่มีคำอธิบายสมเหตุสมผลกำกับ
- **Warning (เตือน):** การจองพื้นที่บน Heap (Dynamic Allocation) โดยไม่จำเป็น, โค้ดส่วนหลักที่ยังขาดชุดการทดสอบครอบคลุม, หรือการตกหล่นข้อมูลคู่มือ API
- **คำแนะนำ:** ไม่ต้องแจ้งเตือนเรื่องรูปแบบโค้ด (formatting) หากผ่านการตรวจสอบสไตล์จากเครื่องมือ `cargo fmt` แล้ว

## ข้อกำหนดการตรวจสอบความถูกต้อง (Verification expectations)
- ฟังก์ชันหรือลอจิกการคำนวณใหม่ทั้งหมดจำเป็นต้องเขียนชุดการทดสอบ (Tests) แนบมาด้วยใน [src/lib_tests.rs](file:///D:/dev/10/poe-sdk-rs-fix-pr10/src/lib_tests.rs) หรือบล็อก `#[cfg(test)]` เสมอ
- หากมีการแก้ไขข้อความหรือรูปแบบของโครงสร้างข้อผิดพลาด จะต้องรันและอัปเดต snapshot test ใน [tests/error_snapshots.rs](file:///D:/dev/10/poe-sdk-rs-fix-pr10/tests/error_snapshots.rs) ร่วมด้วย
- การทดสอบและการรัน Clippy lints ต้องผ่านเรียบร้อยและไร้คำเตือนใด ๆ ทั้งสิ้น
