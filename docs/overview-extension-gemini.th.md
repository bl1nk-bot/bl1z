# เริ่มต้นใช้งานส่วนขยายของ Gemini CLI (Getting started with Gemini CLI extensions)

คู่มือนี้จะแนะนำขั้นตอนการสร้างส่วนขยาย (extension) แรกของคุณสำหรับ Gemini CLI คุณจะได้เรียนรู้วิธีการตั้งค่าส่วนขยายใหม่, การเพิ่มเครื่องมือแบบกำหนดเองผ่านเซิร์ฟเวอร์ MCP, การสร้างคำสั่งแบบกำหนดเอง และการให้บริบทแก่โมเดลด้วยไฟล์ `GEMINI.md`

## สิ่งที่จำเป็นต้องมี (Prerequisites)

ก่อนเริ่มต้น ตรวจสอบให้แน่ใจว่าคุณได้ติดตั้ง Gemini CLI แล้ว และมีความเข้าใจพื้นฐานเกี่ยวกับ Node.js

## ควรใช้เครื่องมือใดเมื่อใด (When to use what)

ส่วนขยายมีหลากหลายวิธีในการปรับแต่ง Gemini CLI

| คุณสมบัติ (Feature) | คืออะไร (What it is) | เมื่อใดที่ควรใช้ (When to use it) | เรียกใช้งานโดย (Invoked by) |
| :------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------- |
| **[เซิร์ฟเวอร์ MCP](/docs/extensions/reference#mcp-servers)** | วิธีมาตรฐานในการเปิดเผยเครื่องมือและแหล่งข้อมูลใหม่ๆ ให้กับโมเดล | ใช้เมื่อคุณต้องการให้โมเดลสามารถ _ทำ_ สิ่งใหม่ๆ ได้ เช่น การดึงข้อมูลจาก API ภายใน, การสืบค้นฐานข้อมูล หรือการควบคุมแอปพลิเคชันภายในเครื่อง นอกจากนี้เรายังรองรับทรัพยากร MCP (ซึ่งสามารถแทนที่คำสั่งแบบกำหนดเองได้) และคำแนะนำระบบ (ซึ่งสามารถแทนที่บริบทแบบกำหนดเองได้) | โมเดล (Model) |
| **[คำสั่งแบบกำหนดเอง](/docs/cli/custom-commands)** | ทางลัด (เช่น `/my-cmd`) ที่รันพรอมต์หรือคำสั่งเชลล์ที่กำหนดไว้ล่วงหน้า | ใช้สำหรับงานที่ทำซ้ำๆ หรือเพื่อบันทึกพรอมต์ที่ยาวและซับซ้อนที่คุณใช้งานบ่อย เหมาะสำหรับการทำงานอัตโนมัติ | ผู้ใช้ (User) |
| **[ไฟล์บริบท (`GEMINI.md`)](/docs/extensions/reference#contextfilename)** | ไฟล์ markdown ที่มีคำแนะนำซึ่งจะถูกโหลดเข้าสู่บริบทของโมเดลเมื่อเริ่มต้นทุกเซสชัน | ใช้เพื่อกำหนด "บุคลิกภาพ" ของส่วนขยายของคุณ, ตั้งค่ามาตรฐานการเขียนโค้ด หรือให้ความรู้พื้นฐานที่โมเดลควรมีอยู่เสมอ | CLI ส่งให้โมเดล |
| **[ทักษะเอเจนต์ (Agent skills)](/docs/cli/skills)** | ชุดคำแนะนำและขั้นตอนการทำงานเฉพาะทางที่โมเดลจะเปิดใช้งานเมื่อจำเป็นเท่านั้น | ใช้สำหรับงานที่ซับซ้อนและเกิดขึ้นเป็นครั้งคราว (เช่น "สร้าง PR" หรือ "ตรวจสอบความปลอดภัย") เพื่อหลีกเลี่ยงความยุ่งเหยิงในหน้าต่างบริบทหลักเมื่อไม่ได้ใช้งานทักษะนั้น | โมเดล (Model) |
| **[Hooks](/docs/hooks)** | วิธีการสกัดกั้นและปรับแต่งพฤติกรรมของ CLI ในเหตุการณ์เฉพาะของวงจรชีวิต (เช่น ก่อน/หลังการเรียกใช้เครื่องมือ) | ใช้เมื่อคุณต้องการทำงานอัตโนมัติตามสิ่งที่โมเดลกำลังทำ เช่น การตรวจสอบพารามิเตอร์ของเครื่องมือ, การบันทึกกิจกรรม หรือการแก้ไขข้อมูลนำเข้า/ส่งออกของโมเดล | CLI |

## ขั้นตอนที่ 1: สร้างส่วนขยายใหม่

วิธีที่ง่ายที่สุดในการเริ่มต้นคือการใช้เทมเพลตที่มีมาให้ เราจะใช้ตัวอย่าง `mcp-server` เป็นพื้นฐาน

รันคำสั่งต่อไปนี้เพื่อสร้างไดเรกทอรีใหม่ชื่อ `my-first-extension` พร้อมกับไฟล์เทมเพลต:

```bash
gemini extensions new my-first-extension mcp-server
```

คำสั่งนี้จะสร้างไดเรกทอรีใหม่ที่มีโครงสร้างดังนี้:

```
my-first-extension/
├── example.js
├── gemini-extension.json
└── package.json
```

## ขั้นตอนที่ 2: ทำความเข้าใจไฟล์ส่วนขยาย

มาดูไฟล์สำคัญในส่วนขยายใหม่ของคุณกัน

### `gemini-extension.json`

นี่คือไฟล์ manifest สำหรับส่วนขยายของคุณ มันจะบอก Gemini CLI ว่าจะโหลดและใช้งานส่วนขยายของคุณอย่างไร

```json
{
  "name": "mcp-server-example",
  "version": "1.0.0",
  "mcpServers": {
    "nodeServer": {
      "command": "node",
      "args": ["${extensionPath}${/}example.js"],
      "cwd": "${extensionPath}"
    }
  }
}
```

- `name`: ชื่อที่ไม่ซ้ำกันสำหรับส่วนขยายของคุณ
- `version`: เวอร์ชันของส่วนขยายของคุณ
- `mcpServers`: ส่วนนี้กำหนดเซิร์ฟเวอร์ Model Context Protocol (MCP) อย่างน้อยหนึ่งเซิร์ฟเวอร์ เซิร์ฟเวอร์ MCP คือวิธีที่คุณสามารถเพิ่มเครื่องมือใหม่ๆ ให้โมเดลใช้งานได้
  - `command`, `args`, `cwd`: ฟิลด์เหล่านี้ระบุวิธีเริ่มต้นเซิร์ฟเวอร์ของคุณ สังเกตการใช้งานตัวแปร `${extensionPath}` ซึ่ง Gemini CLI จะแทนที่ด้วยเส้นทางสมบูรณ์ (absolute path) ไปยังไดเรกทอรีที่ติดตั้งส่วนขยายของคุณ สิ่งนี้ช่วยให้ส่วนขยายของคุณทำงานได้ไม่ว่าจะถูกติดตั้งไว้ที่ใด

### `example.js`

ไฟล์นี้มีซอร์สโค้ดสำหรับเซิร์ฟเวอร์ MCP ของคุณ มันเป็นเซิร์ฟเวอร์ Node.js ง่ายๆ ที่ใช้งาน `@modelcontextprotocol/sdk`

```javascript
/**
 * @license
 * Copyright 2025 Google LLC
 * SPDX-License-Identifier: Apache-2.0
 */

import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import { z } from 'zod';

const server = new McpServer({
  name: 'prompt-server',
  version: '1.0.0',
});

// ลงทะเบียนเครื่องมือใหม่ชื่อ 'fetch_posts'
server.registerTool(
  'fetch_posts',
  {
    description: 'ดึงรายการโพสต์จาก API สาธารณะ',
    inputSchema: z.object({}).shape,
  },
  async () => {
    const apiResponse = await fetch(
      'https://jsonplaceholder.typicode.com/posts',
    );
    const posts = await apiResponse.json();
    const response = { posts: posts.slice(0, 5) };
    return {
      content: [
        {
          type: 'text',
          text: JSON.stringify(response),
        },
      ],
    };
  },
);

// ... (การลงทะเบียนพรอมต์ถูกละไว้เพื่อความกระชับ)

const transport = new StdioServerTransport();
await server.connect(transport);
```

เซิร์ฟเวอร์นี้กำหนดเครื่องมือเดียวชื่อ `fetch_posts` ที่ทำหน้าที่ดึงข้อมูลจาก API สาธารณะ

### `package.json`

นี่คือไฟล์การตั้งค่ามาตรฐานสำหรับโปรเจกต์ Node.js ซึ่งกำหนด dependencies และสคริปต์ต่างๆ

## ขั้นตอนที่ 3: เชื่อมโยงส่วนขยายของคุณ (Link your extension)

ก่อนที่คุณจะสามารถใช้งานส่วนขยายได้ คุณต้องเชื่อมโยงมันกับการติดตั้ง Gemini CLI ของคุณสำหรับการพัฒนาในเครื่อง

1.  **ติดตั้ง dependencies:**

    ```bash
    cd my-first-extension
    npm install
    ```

2.  **เชื่อมโยงส่วนขยาย:**

    คำสั่ง `link` จะสร้าง symbolic link จากไดเรกทอรีส่วนขยายของ Gemini CLI ไปยังไดเรกทอรีการพัฒนาของคุณ ซึ่งหมายความว่าการเปลี่ยนแปลงใดๆ ที่คุณทำจะมีผลทันทีโดยไม่ต้องติดตั้งใหม่

    ```bash
    gemini extensions link .
    ```

คราวนี้ ให้เริ่มต้นเซสชัน Gemini CLI ใหม่ เครื่องมือ `fetch_posts` ใหม่จะพร้อมใช้งาน คุณสามารถทดสอบได้โดยถามว่า: "fetch posts"

## ขั้นตอนที่ 4: เพิ่มคำสั่งแบบกำหนดเอง (Add a custom command)

คำสั่งแบบกำหนดเองเป็นวิธีสร้างทางลัดสำหรับพรอมต์ที่ซับซ้อน มาลองเพิ่มคำสั่งที่ค้นหารูปแบบ (pattern) ในโค้ดของคุณกัน

1.  สร้างไดเรกทอรี `commands` และไดเรกทอรีย่อยสำหรับกลุ่มคำสั่งของคุณ:

    ```bash
    mkdir -p commands/fs
    ```

2.  สร้างไฟล์ชื่อ `commands/fs/grep-code.toml`:

    ```toml
    prompt = """
    กรุณาสรุปสิ่งที่พบสำหรับรูปแบบ `{{args}}`

    ผลการค้นหา:
    !{grep -r {{args}} .}
    """
    ```

    คำสั่งนี้ `/fs:grep-code` จะรับอาร์กิวเมนต์, รันคำสั่งเชลล์ `grep` พร้อมอาร์กิวเมนต์นั้น และส่งผลลัพธ์เข้าไปในพรอมต์เพื่อสรุปข้อมูล

หลังจากบันทึกไฟล์แล้ว ให้เริ่มต้น Gemini CLI ใหม่ คุณสามารถรัน `/fs:grep-code "some pattern"` เพื่อใช้คำสั่งใหม่ของคุณได้ทันที

## ขั้นตอนที่ 5: เพิ่มไฟล์ `GEMINI.md` แบบกำหนดเอง

คุณสามารถให้บริบทแบบถาวรแก่โมเดลได้โดยการเพิ่มไฟล์ `GEMINI.md` ในส่วนขยายของคุณ สิ่งนี้มีประโยชน์สำหรับการให้คำแนะนำแก่โมเดลเกี่ยวกับวิธีการปฏิบัติตัว หรือข้อมูลเกี่ยวกับเครื่องมือของส่วนขยายของคุณ โปรดทราบว่าคุณอาจไม่จำเป็นต้องใช้สิ่งนี้เสมอไปสำหรับส่วนขยายที่สร้างขึ้นเพื่อเปิดเผยคำสั่งและพรอมต์เท่านั้น

1.  สร้างไฟล์ชื่อ `GEMINI.md` ในไดเรกทอรีรากของส่วนขยายของคุณ:

    ```markdown
    # คำแนะนำสำหรับส่วนขยายแรกของฉัน (My First Extension Instructions)

    คุณคือผู้ช่วยนักพัฒนาที่เชี่ยวชาญ เมื่อผู้ใช้ขอให้คุณดึงข้อมูลโพสต์ ให้ใช้เครื่องมือ `fetch_posts` จงกระชับในการตอบสนองของคุณ
    ```

2.  อัปเดตไฟล์ `gemini-extension.json` เพื่อบอกให้ CLI โหลดไฟล์นี้:

    ```json
    {
      "name": "my-first-extension",
      "version": "1.0.0",
      "contextFileName": "GEMINI.md",
      "mcpServers": {
        "nodeServer": {
          "command": "node",
          "args": ["${extensionPath}${/}example.js"],
          "cwd": "${extensionPath}"
        }
      }
    }
    ```

เริ่มต้น CLI ใหม่อีกครั้ง คราวนี้โมเดลจะมีบริบทจากไฟล์ `GEMINI.md` ของคุณในทุกเซสชันที่มีการเปิดใช้งานส่วนขยายนี้

## (ทางเลือก) ขั้นตอนที่ 6: เพิ่มทักษะเอเจนต์ (Agent Skill)

[ทักษะเอเจนต์ (Agent Skills)](/docs/cli/skills) ช่วยให้คุณรวบรวมความเชี่ยวชาญเฉพาะด้านและขั้นตอนการทำงานตามกระบวนการได้ ต่างจาก `GEMINI.md` ที่ให้บริบทแบบถาวร ทักษะจะถูกเปิดใช้งานเมื่อจำเป็นเท่านั้น ซึ่งช่วยประหยัดโทเค็นของบริบท

1.  สร้างไดเรกทอรี `skills` และไดเรกทอรีย่อยสำหรับทักษะของคุณ:

    ```bash
    mkdir -p skills/security-audit
    ```

2.  สร้างไฟล์ `skills/security-audit/SKILL.md`:

    ```markdown
    ---
    name: security-audit
    description:
      ความเชี่ยวชาญในการตรวจสอบโค้ดหาช่องโหว่ด้านความปลอดภัย ใช้เมื่อผู้ใช้ขอให้
      "ตรวจสอบปัญหาความปลอดภัย" หรือ "ตรวจสอบ (audit)" การเปลี่ยนแปลงของพวกเขา
    ---

    # ผู้ตรวจสอบความปลอดภัย (Security Auditor)

    คุณคือนักวิจัยด้านความปลอดภัยที่เชี่ยวชาญ เมื่อตรวจสอบโค้ด:

    1. ค้นหาช่องโหว่ทั่วไป (OWASP Top 10)
    2. ตรวจสอบข้อมูลลับหรือ API keys ที่ถูกฮาร์ดโค้ดไว้
    3. เสนอขั้นตอนการแก้ไขสำหรับสิ่งที่พบ
    ```

ทักษะที่มาพร้อมกับส่วนขยายของคุณจะถูกตรวจพบโดยอัตโนมัติ และโมเดลสามารถเปิดใช้งานได้ในระหว่างเซสชันเมื่อพบงานที่เกี่ยวข้อง

## ขั้นตอนที่ 7: เปิดตัวส่วนขยายของคุณ (Release your extension)

เมื่อคุณพอใจกับส่วนขยายของคุณแล้ว คุณสามารถแบ่งปันกับผู้อื่นได้ สองวิธีหลักในการเปิดตัวส่วนขยายคือผ่าน Git repository หรือผ่าน GitHub Releases การใช้ Git repository แบบสาธารณะเป็นวิธีที่ง่ายที่สุด

สำหรับคำแนะนำโดยละเอียดของทั้งสองวิธี โปรดดูที่ [Extension Releasing Guide](/docs/extensions/releasing)

## บทสรุป (Conclusion)

คุณได้สร้างส่วนขยาย Gemini CLI สำเร็จแล้ว! คุณได้เรียนรู้วิธี:

- สร้างส่วนขยายใหม่จากเทมเพลต
- เพิ่มเครื่องมือแบบกำหนดเองด้วยเซิร์ฟเวอร์ MCP
- สร้างคำสั่งแบบกำหนดเองที่สะดวกสบาย
- ให้บริบทแบบถาวรแก่โมเดล
- รวบรวมทักษะเอเจนต์เฉพาะทาง
- เชื่อมโยงส่วนขยายของคุณสำหรับการพัฒนาในเครื่อง

จากจุดนี้ คุณสามารถสำรวจคุณสมบัติขั้นสูงอื่นๆ และสร้างความสามารถใหม่ๆ ที่ทรงพลังให้กับ Gemini CLI ได้อีกมากมาย
